# Инструкция по деплою на GitHub Pages

## Подготовка репозитория

1. Создайте новый репозиторий на GitHub
2. Инициализируйте git в локальной папке проекта:
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   ```

3. Подключите удаленный репозиторий:
   ```bash
   git remote add origin https://github.com/ваш-username/название-репозитория.git
   git branch -M main
   git push -u origin main
   ```

## Деплой на GitHub Pages

### Способ 1: Через настройки репозитория

1. Перейдите в Settings вашего репозитория на GitHub
2. В боковом меню выберите "Pages"
3. В разделе "Source" выберите ветку `main` и папку `/ (root)`
4. Нажмите "Save"
5. Ваш сайт будет доступен по адресу: `https://ваш-username.github.io/название-репозитория/`

### Способ 2: Через GitHub Actions (опционально)

Можно настроить автоматический деплой при каждом коммите, но для статического сайта это не обязательно.

## После деплоя

После деплоя подождите 1-2 минуты, пока GitHub Pages обработает изменения. Затем откройте ваш сайт по предоставленному адресу.

## Проверка работы

Убедитесь, что:
- ✅ Все страницы открываются корректно
- ✅ Навигация работает
- ✅ Форма валидируется
- ✅ Галерея фильтруется и лайтбокс открывается
- ✅ Адаптивность работает на разных устройствах

## Оптимизация для продакшена

Перед деплоем можно:
1. Минифицировать CSS и JS файлы
2. Оптимизировать изображения (сжать, конвертировать в WebP)
3. Добавить meta теги для SEO
4. Настроить кэширование через .htaccess (для Apache) или аналогичные настройки
